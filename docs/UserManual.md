## 1. 系統簡介

SLSMS 是一套圖書館空間管理示範系統。畫面上會顯示「1樓 · 閱覽 A 區」的 12 個座位 (A1~A4、B1~B4、C1~C4),你可以:

- 即時查看每個座位目前是「空閒」「已預約」「使用中」還是「暫離」
- 模擬讀者預約座位、掃 QR Code 報到、暫時離開、回來、釋放座位
- 用噪音模擬器測試「噪音超標自動推播警示」的功能

---

## 2. 啟動系統

### 方法一:一鍵啟動腳本(建議)

- **Windows**:雙擊 `start.bat`,會自動開啟兩個終端機視窗分別啟動後端與前端
- **macOS / Linux**:在終端機執行
  ```bash
  chmod +x start.sh
  ./start.sh
  ```

腳本會自動安裝前端套件(第一次執行較久),啟動完成後瀏覽器會自動開啟畫面。

### 方法二:手動啟動

1. 開一個終端機,啟動後端:
   ```bash
   cd backend
   mvn spring-boot:run
   ```
   看到後端印出「seeded floor=1F zone=A with 12 seats」代表啟動成功。

2. 開另一個終端機,啟動前端:
   ```bash
   cd frontend
   npm install   # 第一次執行才需要
   npm run dev
   ```

3. 在瀏覽器開啟 `http://localhost:5173`。

### 如何確認系統已連線

畫面右上角會顯示連線狀態:

- **● 後端已連線**(綠色文字):一切正常
- **○ 後端未連線**(灰色文字):請確認後端(Spring Boot)是否已啟動在 `http://localhost:8080`

## 3. 畫面總覽

打開網頁後,從上到下會看到:

1. **標題與圖例**:右上角有四個顏色圖例,分別代表四種座位狀態
   - 🟢 綠色 = 空閒
   - 🟠 橘色 = 已預約
   - 🔴 紅色 = 使用中
   - 🟣 紫色 = 暫離

2. **統計卡片**:顯示目前「空閒 / 已預約 / 使用中 / 暫離」各有幾個座位

3. **樓層座位圖**:12 個座位方格,點擊任一座位可選取它(會以外框標示)

4. **操作面板**:選取座位後,這裡會顯示該座位目前可以執行的動作按鈕

5. **噪音模擬器**:模擬 A 區的噪音感測器

6. **近期警示清單**:顯示最近發生的噪音警示紀錄

## 4. 座位操作流程

座位狀態會依照下圖規則轉換,**每個狀態只能執行特定的動作**——這是系統背後「State Pattern」設計的展現,操作面板只會顯示「目前合法」的按鈕。

完整示範流程(以座位 A1 為例):

### 步驟 1:預約座位

1. 點擊座位 **A1**(此時應為綠色「空閒」)
2. 在下方操作面板點擊 **「預約」**
3. A1 變成橘色,狀態顯示為「已預約」

### 步驟 2:掃 QR Check-in

1. 再次點擊 A1,在操作面板點擊 **「掃 QR Check-in」**
2. 畫面會彈出一個 QR Code 卡片
3. 點擊 **「模擬掃描成功」**(這代表「讀者實際掃描了座位上的 QR Code」)
4. A1 變成紅色,狀態顯示為「使用中」,並會標示使用者 ID

> 若想取消,可以點擊 QR Code 卡片旁的「取消」按鈕關閉視窗,不會改變座位狀態。

### 步驟 3:暫時離開

1. 選取 A1(目前為「使用中」),點擊 **「暫時離開」**
2. A1 變成紫色,狀態顯示為「暫離」

### 步驟 4:回來

1. 選取 A1(目前為「暫離」),點擊 **「回來」**
2. A1 變回紅色,狀態顯示為「使用中」

### 步驟 5:釋放座位

1. 選取 A1,點擊 **「釋放」**
2. A1 變回綠色,狀態顯示為「空閒」,可供下一位讀者使用

### 各狀態可以做什麼

| 座位目前狀態 | 可以點的按鈕 |
|---|---|
| 🟢 空閒 | 預約、掃 QR Check-in |
| 🟠 已預約 | 掃 QR Check-in、釋放 |
| 🔴 使用中 | 暫時離開、釋放 |
| 🟣 暫離 | 回來、釋放 |

> 💡 畫面每 2 秒會自動向後端更新一次座位狀態,所以如果有其他人(或其他瀏覽器分頁)同時操作,你的畫面也會在幾秒內反映最新狀態。

## 5. 噪音模擬器

噪音模擬器位於頁面下方的「A 區噪音感測器」卡片,用來示範「噪音超標時系統會自動推播警示」的功能。

### 操作方式

- **「+5 dB」**:增加 5 分貝
- **「−5 dB」**:減少 5 分貝
- **「+20 dB (突發)」**:模擬一次突發大聲響(例如有人大聲講話)
- **「重置」**:把噪音值重設回預設值

### 觀察重點

A 區的噪音門檻是 **65 dB**:

- 噪音數值在門檻以下時,顯示為**綠色**
- 接近門檻(門檻前 10 dB 以內)時,顯示為**橘色**
- **超過 65 dB** 時,數值變成**紅色**,並且畫面上方會出現一條**會閃爍(脈動)的紅色警示條**,內容類似:
  > ⚠ Observer 觸發 · A 區噪音超標,後端已推送通知至管理員 + 數位看板

- 同時,頁面最下方的「近期警示」清單會新增一筆紀錄,記錄這次超標事件

### 小提醒

- 系統採用「邊緣觸發」設計:只有在「從未超標 → 超標」的那一瞬間才會推播一次警示,**持續維持在超標狀態不會重複跳出警示**。
- 想再次觸發警示,可以先點「−5 dB」把數值降回門檻以下,再點「+20 dB」讓它重新跨越門檻。

## 6. 錯誤訊息說明(紅色提示條)

如果你點擊了一個**目前狀態不允許的動作**,畫面上方會出現一條**紅色錯誤提示條**,例如:

> `Action 'reserve' is not allowed in state 'OCCUPIED'`

這代表:**這個座位現在的狀態,不能執行你剛剛點的這個動作。**

常見情境:

- 座位已經「使用中」,卻嘗試再次「預約」→ 不允許
- 座位是「空閒」,卻嘗試「釋放」或「暫時離開」→ 不允許(因為根本沒人在用)

這個提示條正是系統背後「State Pattern」在保護資料一致性:每個狀態只接受合理的操作,其他操作會被系統拒絕並顯示原因。

> 如果出現「找不到座位」之類的錯誤,代表你操作的座位 ID 在系統中不存在,請重新整理頁面後再試一次。

## 7. 目前版本的限制

為了讓 demo 聚焦在核心設計模式(State Pattern、Observer Pattern)上,Version 1.0 刻意簡化了以下部分,使用時請留意:

- **沒有登入機制**:所有操作都以同一個示範使用者身分(`demo-user-001`)進行,目前無法切換「讀者」與「管理員」身分。
- **資料不會永久保存**:座位狀態存在記憶體中,**後端重新啟動後,所有座位會回到初始狀態(全部空閒)**。
- **QR Code 為模擬畫面**:點擊「掃 QR Check-in」會顯示一個示意用的 QR 圖片,並不需要真的用手機掃描;「模擬掃描成功」按鈕等同於完成掃描動作。
- **沒有自動逾時釋放**:「已預約 15 分鐘未報到自動釋放」「暫離 30 分鐘自動釋放」這兩個邏輯雖然在程式內部已實作並通過測試,但 V1.0 demo 中**不會自動倒數觸發**,需要由下一版本(V2)補上自動排程機制。
- **沒有報表功能**:目前看不到「使用率報表」或「CSV 匯出」,這部分規劃在 Version 2.0。
- **只有噪音感測,沒有溫度感測**:噪音模擬器是真實運作的,溫度感測為 Version 2.0 規劃項目。
